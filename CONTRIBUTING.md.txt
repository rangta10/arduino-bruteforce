# Contributing to Arduino Brute Force Demo

Thank you for considering contributing to this educational security project! We welcome contributions that improve the educational value and security awareness aspects of this demonstration.

## Code of Conduct

This project is intended for **educational purposes only**. By contributing, you agree to:

- Use this knowledge responsibly
- Not promote or facilitate unauthorized access to systems
- Help improve security awareness and education
- Be respectful and constructive in discussions

## How to Contribute

### Reporting Bugs

If you find a bug, please open an issue with:

1. **Clear title** describing the problem
2. **Steps to reproduce** the issue
3. **Expected behavior** vs actual behavior
4. **Environment details** (OS, Python version, Arduino board)
5. **Screenshots** if applicable

### Suggesting Enhancements

We welcome ideas for:

- Additional security demonstrations
- Improved documentation
- Better user experience
- Performance optimizations
- Educational content improvements

### Pull Requests

1. **Fork** the repository
2. **Create a branch** (`git checkout -b feature/your-feature`)
3. **Make your changes** with clear commits
4. **Test thoroughly** on your system
5. **Update documentation** if needed
6. **Submit pull request** with detailed description

- Use clear, concise language
